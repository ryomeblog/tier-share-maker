# Tier List Maker — アプリケーション仕様書

**Version:** 1.0  
**更新日:** 2026年3月11日  
**技術基盤:** Next.js + Cloudflare Pages

---

## 1. プロジェクト概要

Webブラウザ上で動作するTier表作成アプリケーション。ユーザーは画像URL またはローカル画像ファイルを用いてアイテムを追加し、S〜Eランクにドラッグ＆ドロップで配置できる。作成したTier表はURL共有・画像保存・OGPプレビューに対応する。

### 1.1 技術スタック

| レイヤー | 技術 | 備考 |
|---|---|---|
| フレームワーク | Next.js (App Router) | React 18+ / TypeScript |
| スタイリング | Tailwind CSS | ユーティリティファースト |
| D&D | dnd-kit | ドラッグ＆ドロップライブラリ |
| 画像キャプチャ | html-to-image | DOM→PNG変換 |
| URL圧縮 | lz-string | データ圧縮エンコード |
| OGP画像生成 | Satori + @resvg/resvg-wasm | Cloudflare Workers上で実行 |
| デプロイ | Cloudflare Pages | Edge Functions対応 |

## 2. 機能要件

### 2.1 Tier表の基本構造

デフォルトTier構成は S / A / B / C / D / E の6段階。各Tierはラベル名と背景色を持つ。

| Tier | デフォルト名 | 背景色 | 用途 |
|---|---|---|---|
| S | S | `#FF6B6B` | 最高ランク |
| A | A | `#FFA94D` | 優秀 |
| B | B | `#FFD43B` | 良好 |
| C | C | `#69DB7C` | 普通 |
| D | D | `#74C0FC` | 低め |
| E | E | `#B197FC` | 最低ランク |

**カスタマイズ:** Tier名のインライン編集、E以降のTier追加（F, G等）、不要なTierの削除が可能。

### 2.2 アイテム追加

ユーザーは以下の2つの方法でアイテム（画像）をTier表に追加できる。

| 追加方法 | 説明 | 共有時の動作 |
|---|---|---|
| 画像URL入力 | 外部URLを貼り付けて追加 | URLがそのまま共有される |
| ローカル画像 | 端末から画像ファイルをアップロード | **共有URLには含まれない** |

#### ローカル画像の取り扱い

- アップロードされた画像はブラウザの `Object URL`（`blob:〜`）または `Base64 Data URL` としてメモリ上に保持する
- 編集中は通常通り表示・配置が可能
- **URL共有時にはローカル画像はエンコード対象から除外する**
- ローカル画像を含むアイテムには視覚的なバッジ（例: 🔒アイコン）を表示し、「この画像は共有URLに含まれません」とユーザーに明示する
- 「URLをコピー」実行時にローカル画像が存在する場合、「ローカル画像○件は共有URLに含まれません。画像として保存すればすべてのアイテムを共有できます。」と通知する
- 画像エクスポート（PNG保存）ではローカル画像も含めてキャプチャされる

### 2.3 ドラッグ＆ドロップ操作

- 未配置エリア（プール）から各Tier行へのドラッグ
- Tier行内での並び替え（左右の順序変更）
- Tier間の移動（例: AランクからSランクへ）
- Tier行からプールへの戻し
- タッチデバイス対応（ロングプレスでドラッグ開始）

### 2.4 Tierカスタマイズ

| 操作 | 説明 | UI |
|---|---|---|
| Tier名変更 | ラベルをクリックでインライン編集 | テキスト入力 |
| Tier追加 | 一番下に新しいTierを追加 | 「+ Tier追加」ボタン |
| Tier削除 | 指定のTierを削除（アイテムはプールに戻る） | ×ボタン |
| 背景色変更 | 各Tierの背景色をカラーピッカーで変更 | カラーパレット |

## 3. URL共有機能

### 3.1 データエンコード方式

Tier表の状態をサーバーレスで共有するため、URLのフラグメント（`#`以降）に圧縮データを格納する。フラグメントはサーバーに送信されないため、データはクライアント側のみで処理される。

### 3.2 エンコードフロー

1. **ステート取得:** Tier名・背景色・各Tierのアイテム（画像URL + ラベル）をJSON化。ローカル画像アイテムは除外する
2. **lz-string圧縮:** `compressToEncodedURIComponent()` でURI安全な文字列に変換
3. **URL生成:** `https://example.com/#<圧縮データ>` 形式で生成

### 3.3 デコードフロー

1. **URLフラグメント取得:** `window.location.hash` から圧縮データを抽出
2. **lz-string展開:** `decompressFromEncodedURIComponent()` でJSON文字列に復元
3. **ステート復元:** `JSON.parse` でTier表の状態を復元し描画

### 3.4 共有データ構造（JSON）

```json
{
  "title": "My Tier List",
  "tiers": [
    {
      "id": "tier-s",
      "name": "S",
      "color": "#FF6B6B",
      "items": [
        {
          "id": "item-1",
          "url": "https://example.com/image.png",
          "label": "アイテム名"
        }
      ]
    }
  ],
  "pool": []
}
```

| フィールド | 型 | 説明 |
|---|---|---|
| title | string | Tier表のタイトル |
| tiers | TierRow[] | Tier行の配列 |
| tiers[].id | string | Tierの一意識別子 |
| tiers[].name | string | Tier名（例: S, A, B） |
| tiers[].color | string | 背景色（HEX） |
| tiers[].items | Item[] | 配置されたアイテム |
| tiers[].items[].id | string | アイテムの一意識別子 |
| tiers[].items[].url | string | 画像URL |
| tiers[].items[].label | string? | アイテムのラベル（任意） |
| pool | Item[] | 未配置アイテム |

**URL長制限:** 実用上の上限約8,000文字。圧縮後のデータがこれを超える場合、ユーザーに「アイテムが多すぎます。画像として保存してください」と警告を表示する。

## 4. 画像保存・共有機能

### 4.1 クライアント側画像生成

html-to-imageライブラリを使用し、Tier表のDOM要素をPNG画像に変換する。

1. **エクスポート対象:** Tier表全体のDOM要素（ヘッダー・全Tier行・フッター）
2. **出力形式:** PNG（透過背景なし、白背景）
3. **ダウンロード:** 自動でブラウザのダウンロードをトリガー
4. **ローカル画像:** PNG保存時にはローカル画像も含めてキャプチャされる

### 4.2 CORSプロキシAPI

外部画像のhtml-to-imageでの取得失敗を回避するため、Cloudflare Functionsでプロキシエンドポイントを用意する。

| 項目 | 内容 |
|---|---|
| エンドポイント | `/api/proxy-image?url=<エンコード済みURL>` |
| メソッド | GET |
| レスポンス | 画像バイナリ（Content-Typeをそのまま転送） |
| レート制限 | リクエストあたり最大5MB、同一IPから毎分60リクエスト |
| キャッシュ | `Cache-Control: public, max-age=86400` |

## 5. OGPプレビュー機能

### 5.1 概要

SNSやメッセンジャーでURLを共有した際、Tier表のプレビュー画像が自動生成されて表示される。

### 5.2 実装方式

Cloudflare Functionsのサーバーサイド処理で、リクエスト時にUser-Agentを判定する。SNSクローラー（Twitter/X, LINE, Discord等）からのアクセスの場合、OGPタグを含むHTMLを返却する。通常のブラウザアクセスの場合は通常のSPAを返す。

### 5.3 OGP画像生成フロー

1. URLのフラグメントからTierデータを取得・展開（※フラグメントはサーバーに届かないため、クエリパラメータ `?data=` も併用する）
2. Satoriで JSXテンプレートからSVGを生成
3. @resvg/resvg-wasmでSVGをPNGに変換
4. レスポンスとしてPNGを返却（Cache-Control付き）

> **注意:** フラグメント（`#`以降）はサーバーに送信されない。そのため、共有URL生成時にはフラグメントと同じデータをクエリパラメータ `?data=<圧縮データ>` としても付与する。OGPエンドポイントはこのクエリパラメータを読み取る。

### 5.4 OGPメタタグ

| タグ | 値 |
|---|---|
| og:title | Tier表のタイトル（未設定時: 「Tier List」） |
| og:description | 「Tier List Makerで作成」 |
| og:image | `/api/og?data=<圧縮データ>` |
| og:image:width | 1200 |
| og:image:height | 630 |
| twitter:card | summary_large_image |

### 5.5 OGP画像デザイン

1200×630pxの画像に以下をレンダリングする。

- タイトルテキスト（上部）
- 各Tier行（ラベル + アイテムサムネイル、最大8個まで表示）
- 「Tier List Maker」ブランドロゴ（下部）

**注意:** OGP画像では外部画像URLの取得が必要。Cloudflare Workers内で `fetch()` を使って画像を取得し、Base64エンコードしてSatoriに渡す。

## 6. ページ構成

| パス | 説明 | 実装 |
|---|---|---|
| `/` | Tier表エディター（メインページ） | Client Component |
| `/api/og` | OGP画像生成エンドポイント | Cloudflare Function |
| `/api/proxy-image` | CORSプロキシエンドポイント | Cloudflare Function |

**SPA構成:** アプリ全体はシングルページのSPA。URLのフラグメントにより編集モードと閲覧モードを切り替える。フラグメントにデータがある場合は閲覧モード（共有されたTier表の表示）、ない場合は新規作成の編集モードとなる。

## 7. UI/UX設計

### 7.1 レイアウト

| エリア | 内容 |
|---|---|
| ヘッダー | アプリタイトル + Tier表タイトル編集 + アクションボタン群 |
| Tierエリア | 各Tier行（ラベル + アイテムグリッド） |
| プールエリア | 未配置アイテム置き場 + アイテム追加UI |
| フッター | コピーライト + リンク |

### 7.2 アクションボタン

- **「URLをコピー」**: 共有用URLをクリップボードにコピー。ローカル画像が含まれる場合は通知を表示
- **「画像として保存」**: Tier表をPNG画像としてダウンロード（ローカル画像含む）

### 7.3 アイテム追加UI

プールエリア内に以下の2つの追加方法を提供する。

- **「URL追加」タブ:** テキストフィールドに画像URLを貼り付けて追加。プレビュー表示後に確定
- **「画像アップロード」タブ:** ファイル選択ボタン、またはドラッグ＆ドロップでローカル画像を追加。複数ファイル同時選択に対応

### 7.4 レスポンシブ対応

- **デスクトップ:** Tier行は横幅100%、アイテムは折り返し表示
- **モバイル:** アイテムサイズを縮小、タッチドラッグ対応
- **ブレークポイント:** 768pxでレイアウト切り替え

## 9. ステート管理

`useReducer` を使用した集約的なステート管理。以下のアクションをサポートする。

| アクション | 説明 |
|---|---|
| ADD_ITEM | プールに新しいアイテムを追加（URL or ローカル） |
| REMOVE_ITEM | アイテムを削除 |
| MOVE_ITEM | アイテムを別のTier/位置に移動 |
| ADD_TIER | 新しいTier行を追加 |
| REMOVE_TIER | Tier行を削除（アイテムはプールへ） |
| RENAME_TIER | Tier名を変更 |
| CHANGE_TIER_COLOR | Tierの背景色を変更 |
| SET_TITLE | Tier表タイトルを変更 |
| RESET | 初期状態にリセット |
| LOAD_FROM_URL | URLデータから状態を復元 |

### 内部データ構造

ステート内部では、各アイテムに `source` フィールドを持たせてURLとローカルを区別する。

```typescript
type ItemSource = "url" | "local";

interface Item {
  id: string;
  url: string;       // 外部URLまたはblob:/data: URL
  label?: string;
  source: ItemSource; // "url" = 外部URL, "local" = ローカルファイル
}
```

URL共有時のJSONエンコードでは `source: "local"` のアイテムをフィルタリングして除外する。

## 10. Cloudflare Pagesデプロイ設定

### 10.1 ビルド設定

| 項目 | 値 |
|---|---|
| フレームワークプリセット | Next.js |
| ビルドコマンド | `npx @cloudflare/next-on-pages` |
| 出力ディレクトリ | `.vercel/output/static` |
| Node.jsバージョン | 18.x |

### 10.2 @cloudflare/next-on-pages

Next.jsのApp Router + Cloudflare Pages Functionsの統合には `@cloudflare/next-on-pages` を使用する。これによりRoute Handlers（`/api/og`, `/api/proxy-image`）がCloudflare Workersとして動作する。

### 10.3 wasmファイルの取り扱い

`@resvg/resvg-wasm` はWASMバイナリを使用するため、Cloudflare Workersのバンドルサイズ制限（無料プラン: 1MB）に注意が必要。必要に応じてWorkersの有料プラン（10MB）へのアップグレードを検討する。

## 11. セキュリティ考慮事項

- **プロキシAPIの悪用防止:** リクエストレート制限、対象URLのバリデーション（画像拡張子の確認、httpsのみ許可）
- **XSS対策:** URLから読み込んだデータのサニタイズ、画像URLのhttpsのみ許可
- **CSPヘッダー:** `img-src` に信頼できるドメインのみ許可、またはワイルドカード（`*`）で全許可（トレードオフを検討）
- **入力バリデーション:** Tier名の文字数制限（20文字）、アイテム数上限（100個）、画像URL長制限（2,000文字）
- **ローカル画像:** `blob:` / `data:` URLはサニタイズ不要だが、ファイルサイズ上限（5MB/枚）を設ける
